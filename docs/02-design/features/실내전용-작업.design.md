# 실내 전용 작업 기록 및 협상 가격 입력 기능 — Design

- Feature: 실내전용-작업
- Phase: Design
- Date: 2026-04-12

---

## Context Anchor

| 항목 | 내용 |
|------|------|
| **WHY** | 실내 전용 작업이 앱 밖에서 거래되면 정산 불투명, 직원 급여 산정도 어려워짐 |
| **WHO** | 직원(작업 완료 처리), 사장(청구 확인 및 관리) |
| **RISK** | 직원이 협상 가격을 잘못 입력할 위험 → 사장 확인 단계 필요 |
| **SUCCESS** | 실내 전용 작업도 app에서 완료 처리, 청구서에 자동 포함 |
| **SCOPE** | CompletionModal 수정 + 관리자 UI 배지 표시 + DB 스키마 무변경 |

## 1. 개요

실내만 작업하는 케이스를 앱에 기록하고, 협상 가격(기본 20,000원)을 직접 입력할 수 있는 기능.

### 1.1 작업 유형 확장

```
service_type 필드 값:
- 'regular' (기존): 외부 세차만 → 차량 단가
- 'interior' (기존): 외부+실내 → 단가 + 10,000원
- 'interior_only' (신규): 실내만 → 직접 입력 (기본 20,000원)
```

## 4. 컴포넌트 설계

### 4.1 CompletionModal — 작업 유형 선택

```
[작업 유형]
● 외부 세차        → 가격: 차량.unit_price
○ 외부 + 실내      → 가격: 차량.unit_price + 10,000
○ 실내 전용        → 가격: [입력 필드] (기본 20,000)

실내 선택 시:
- 입력 필드: interiorOnlyPrice (number, min=0, step=1000)
- 기본값: 20,000
- UI: 주황색 배경 (경고 주의)
```

### 4.2 UI 플로우

1. 직원이 세차 완료 버튼 클릭
2. CompletionModal 열기
3. 작업 유형 선택 (라디오 3개)
4. 실내 전용 선택 시 가격 입력 필드 노출
5. 당번 선택 → 시각 입력 → 완료 처리
6. API 호출: `POST /api/wash-records/create`
7. Supabase: `wash_records` 저장 (service_type='interior_only', price=협상가)

## 5. API 계약

### 5.1 POST /api/wash-records/create

**요청본문:**
```json
{
  "vehicle_id": "uuid",
  "schedule_id": "uuid or null",
  "wash_date": "2026-04-12",
  "price": 20000,
  "service_type": "interior_only",
  "worker_id": "uuid",
  "worked_by": "worker",
  "completed_by": "직원명",
  "completed_at": "2026-04-12T15:30:00",
  "memo": "고객 협상으로 실내만"
}
```

**응답:**
```json
{
  "success": true,
  "data": { "id": "uuid", ... },
  "message": "세차 완료 처리되었습니다."
}
```

## 10. 위험 & 대응

| 위험 | 대응 |
|------|------|
| 직원이 0원 입력 | 가격 > 0 유효성 검사 추가 |
| 기존 데이터 호환성 | service_type='interior'는 기존 그대로, 신규만 'interior_only' |
| 급여 산정 영향 | interior_only도 작업 건수에 포함, 단가 기준은 자동 처리 |

## 1. 현황 분석

### 현재 billings 테이블 구조

```sql
billings (
  paid_amount    INTEGER,         -- 입금액 ✓
  payment_status TEXT,            -- paid/partial/unpaid ✓
  sent_at        TIMESTAMPTZ,     -- 카톡 발송 시간 ✓ (미표시)
  -- 아래가 빠져 있음:
  -- paid_at        (입금 날짜)  ← 없음
  -- payment_method (증빙 종류)  ← 없음
)
```

### 현재 부족한 것

| 항목 | 현황 | 목표 |
|------|------|------|
| 입금 날짜 | 없음 | `paid_at` 컬럼 추가 + UI |
| 증빙 종류 | 없음 | `payment_method` 컬럼 추가 + UI |
| 카톡 발송 날짜 | DB에 있으나 화면에 안 보임 | `sent_at` 표시 추가 |
| 금액 표시 | "45,000원" | "45000" 숫자만 (복붙용) |

## 2. 요구사항

### R1. 입금 날짜 기록 (HIGH)

- **입금완료 버튼** 클릭 시 → 날짜 선택 팝업 (기본값: 오늘)
- 날짜 선택 후 DB에 `paid_at` 저장
- 청구현황에 "4/12 입금" 형태로 표시

### R2. 증빙 종류 선택 (HIGH)

결제 수단 선택:

| 값 | 표시 | 의미 |
|----|------|------|
| `cash` | 현금 | 현금 수령 |
| `transfer` | 계좌이체 | 계좌 이체 |
| `card` | 카드 | 신용/체크카드 |
| `cash_receipt` | 현금영수증 | 현금 + 영수증 발급 |

> 입금 처리 모달에서 날짜 + 증빙 종류를 한 번에 선택

### R3. 카톡 발송 날짜·시간 표시 (HIGH)

- `billings.sent_at` 이미 DB에 존재하나 화면에 표시 안됨
- 카드에 "카톡 미발송" 또는 "4/1 09:30 발송" 형태로 표시 추가

### R4. 금액 복붙용 숫자 표시 (MEDIUM)

- 기존: "45,000원" → 신규: 금액 카드 옆에 숫자만 복사 버튼
- 또는 세금 신고용 내보내기(CSV) 버튼 추가

### R5. 카카오 비즈니스 메시지 연동 (MEDIUM — 준비 단계)

- 현재: 카톡 텍스트 복사 후 수동 발송
- 목표: 버튼 1번으로 알림톡 자동 발송 + `sent_at` 자동 기록

## 4. UI 변경 계획

### 4.1 입금 처리 — 개선

**현재 (Before):**
```
[입금완료] 버튼 → 즉시 paid 처리
```

**개선 (After):**
```
[입금완료] 버튼 → 입금 정보 팝업
  입금 날짜: [ 2026-04-12 ] ← 날짜 선택기
  결제 수단: [○ 현금] [○ 계좌이체] [○ 카드] [○ 현금영수증]
  [확인]
  → paid 처리 + paid_at + payment_method 저장
```

### 4.2 청구 카드 — 표시 개선

```
┌──────────────────────────────────────┐
│ 홍길동                 미입금 →  입금완료 │
│ 소나타 12가3456                        │
│                                      │
│ 청구: 45,000   ← 숫자만+복사 버튼       │
│ 증빙: —        ← 입금 후 "계좌이체" 표시 │
│ 입금일: —      ← 입금 후 "4/12" 표시   │
│ 카톡: 4/1 09:30 발송  ← sent_at 표시  │
└──────────────────────────────────────┘
```

### 4.3 세금 신고용 표 (목록 하단)

월별 필터 기준, 표로 표시:

| 입금날짜 | 고객명 | 금액 | 증빙 | 카톡발송 |
|---------|--------|------|------|---------|
| 4/12 | 홍길동 | 45000 | 계좌이체 | 4/1 |
| 4/10 | 김철수 | 35000 | 현금 | 4/1 |
| ... | ... | ... | ... | ... |

- 클립보드 복사 버튼 (엑셀 붙여넣기용)
- 숫자에 "원" 표시 없음

## 5. 카카오 비즈니스 메시지 — 방법 선택

### 방법 비교

| 방법 | 난이도 | 비용 | 제한 |
|------|--------|------|------|
| **A. 현재 방식** (텍스트 복사 후 수동 발송) | 없음 | 없음 | 수동 |
| **B. 카카오 알림톡 API** | ★★★★ | 건당 7원 내외 | 템플릿 심사 2-4주 |
| **C. 문자 SMS API** (쿨에스엠에스 등) | ★★ | 건당 9원 | 발신번호 등록 |
| **D. 카톡 링크 URL 자동 생성** | ★ | 없음 | 수동 클릭 필요 |

### 추천 로드맵

**1단계 (지금): 방법 D — 카톡 링크 개선**
- 현재 복사 텍스트를 카카오톡 나에게 보내기 링크로 개선
- 버튼 클릭 → 카카오앱 열기 → 직접 고객에게 전달
- `sent_at` 수동 기록 버튼 별도 추가 ("발송 완료" 체크)

**2단계 (1-2개월 후): 방법 B — 알림톡 API**
- 카카오 비즈니스 채널 개설 필요 (사장님 직접)
- 메시지 템플릿 신청 (심사 2-4주)
- API 키 발급 → 앱에 연동
- 구현 완료 시: 버튼 1번 → 자동 발송 + sent_at 자동 기록

### 알림톡 준비 필요 사항 (사장님 직접)

1. **카카오 비즈니스 채널 만들기**
   - kakao.com/biztools 접속
   - "채널 개설" → 세차 비즈니스 채널 등록

2. **발신 프로필 등록**
   - 채널 이름: 예) "새차처럼세차"

3. **메시지 템플릿 신청**
   - 예시 템플릿:
     ```
     [새차처럼] #{월}월 세차 청구 안내
     고객명: #{고객명}님
     청구금액: #{금액}원
     납부 기한: #{마감일}
     문의: #{전화번호}
     ```
   - 심사 완료 후 앱에서 자동 발송 가능

## 7. 구현 순서 (추천)

1. **DB 마이그레이션** — `paid_at`, `payment_method` 컬럼 추가 (5분)
2. **입금 처리 모달 개선** — 날짜 + 증빙 선택 추가 (1시간)
3. **청구 카드 표시 개선** — paid_at, payment_method, sent_at 표시 (30분)
4. **세금 신고용 표** — 하단에 리스트 추가, 숫자 복사 버튼 (30분)
5. **카톡 발송 완료 수동 체크 버튼** — sent_at 기록 (15분)

**카카오 알림톡 자동 발송**은 채널 개설 및 심사 완료 후 별도 진행

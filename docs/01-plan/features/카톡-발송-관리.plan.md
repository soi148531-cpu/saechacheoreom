## 1. 요구사항

### 1.1 기능 요구사항

#### 1단계: 청구현황 페이지 (즉시)
1. **DB 스키마 확장**
   - `billings` 테이블에 `message_sent_at TIMESTAMPTZ NULL` 컬럼 추가
   - 기존 컬럼: `paid_at`, `payment_method` (세금신고용, 이미 추가됨)
   - 새 컬럼 목적: 카톡 발송 시각 기록

2. **청구 카드 UI**
   - 발송 상태 배지 표시 (현재 3가지 상태)
     - `message_sent_at IS NULL` → "카톡 미발송" (빨강)
     - `message_sent_at IS NOT NULL` → "발송완료 YYYY-MM-DD 13:45" (회색)
   - 기존 배지 아래에 추가

3. **필터 탭 추가**
   ```
   [전체] [발송완료] [미발송]
   ```
   - 기존 결제수단 필터(`결제상태-세금신고-청부현황`에서)와 독립적
   - 고객이 두 필터를 조합해도 작동

4. **"카톡 발송" 버튼**
   - 위치: 청구 카드에 "입금완료" 버튼 옆 (또는 우측)
   - 클릭 → `message_sent_at = NOW()`로 Supabase 업데이트
   - 기능: 실제 카톡 발송 안 함 (아직) → 발송 상태만 기록, 메시지 본문은 DB 템플릿에서 가져옴
   - 텍스트: "카톡 발송" 또는 "재발송"

5. **발송 기록 관리**
   - 카톡 발송 버튼 항상 활성화 (중복 발송/재발송 가능)
   - 클릭할 때마다 `message_sent_at` 갱신 (최신 발송 시각 기록)
   - 이미 발송된 항목 → "재발송" 텍스트로 표시

6. **메시지 템플릿 관리** (NEW)
   - 설정 페이지에서 메시지 본문 수정 가능 (배포 불필요)
   - 메시지는 DB 테이블(`message_templates`)에서 관리
   - 변수 자동 치환: `{customer_name}`, `{amount}`, `{car_name}`, `{unit_number}`, `{month}`

#### 2단계: 이력조회 대시보드 (로드맵)
- 청구현황 데이터 기반
- 월별 카톡 발송 현황 리포트
- "카톡 미발송" 항목 필터 및 조회

### 1.2 기술 요구사항

- **DB**: Supabase PostgreSQL (migration via `scripts/apply-migration.mjs`)
  - `billings` 테이블: `message_sent_at` 컬럼
  - `message_templates` 테이블: 메시지 템플릿 저장소 (배포 없이 수정 가능)
- **Frontend**: React Client Component, Supabase 클라이언트
- **메시지 관리**: 템플릿 + 변수 치환 (고객명, 금액, 차량명 등)
- **설정 페이지**: 메시지 템플릿 수정 UI
- **타입**: `Billing` 인터페이스에 `message_sent_at: string | null` + `MessageTemplate` 타입 추가

### 1.3 제외 사항 (Out of Scope)

- 카카오 API 연동 (2단계 이후)
- 실제 메시지 발송 (아직)
- 메시지 발송 이력 (단일 타임스탬프만, 여러 회차 기록 X)
- 재발송 스케줄링

**파일**: `supabase/migrations/15_message_sent_at.sql`

## 6. 주요 결정사항

1. **DB 스키마**
   - `message_sent_at TIMESTAMPTZ NULL` (따로 테이블 안 만들고 기존 billings에 추가)
   - 이유: 청구 건마다 1:1 대응, 정규화 불필요

2. **필터 위치**
   - 기존 결제상태 필터와 독립적 (중첩 가능)
   - 이유: 사용자가 "미발송 항목 중 입금완료된 것" 같은 조합 조회 가능

3. **발송 상태 기록**
   - "카톡 발송" 버튼 클릭 시 `NOW()` 저장
   - 실제 카톡 발송은 2단계 이후 (아직 API 없음)

4. **UI 배치**
   - 기존 버튼("입금완료", "결제수단") 옆에 추가
   - 혼잡을 피해 필요시 "더보기" 메뉴로 이동 가능
